# Contributing Guidelines

Thank you for your interest in contributing to this project.

This repository was created as an end-to-end machine learning and supply chain forecasting portfolio project. Contributions, suggestions, and improvements are welcome.

## How to Contribute

If you would like to contribute, please follow these guidelines:

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test your changes locally
5. Submit a pull request with a clear description

## Contribution Areas

Potential contribution areas include:

- data pipeline improvements
- feature engineering enhancements
- model optimization
- API improvements
- evaluation metrics
- documentation and examples
- visualization improvements

## Coding Standards

Please follow these basic rules:

- Use clear, readable Python code
- Prefer modular functions over long scripts
- Add comments where logic may not be obvious
- Use meaningful file and variable names
- Keep the project structure organized

## Testing

Before submitting changes, make sure that:

- scripts run successfully from the project root
- file paths remain consistent
- no raw datasets or large model artifacts are committed
- documentation is updated when relevant

## Data Usage

This project uses the M5 Forecasting dataset from Kaggle.

Due to dataset licensing and repository size considerations:

- raw Kaggle data should not be committed to the repository
- processed datasets should also remain local unless explicitly needed
- contributors should follow the dataset instructions in the README

## Pull Request Notes

When opening a pull request, please include:

- a short summary of the change
- why the change improves the project
- any impact on outputs, reports, or APIs

## Questions or Suggestions

If you would like to suggest an improvement, feel free to open an issue or submit a pull request.
